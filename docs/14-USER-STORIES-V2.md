## 0. THE IA FIX FIRST (the «нажал на строку — попал в цифровую кашу» problem)

Root cause: drill-down renders raw metrics without narrative or hierarchy. The fix is a **pattern, not a page**: every click lands on a **Паспорт** (passport) layout with strict order:

1. **Кто я** — plain-language header sentence: «Строка договора: МРТ · КДУ · ОСМС · план на год 38,4 млн ₸ · договор №…, ред. доп. согл. №3».
2. **Вердикт** — one black-block verdict: «ҚАУІП: асыра орындау / Риск перевыполнения. Объём закончится 14.10.2026.»
3. **Почему** — forecast chart + the explanation sentence + 3 labeled KPIs (each with unit, period, «как посчитано» popover from metrics.yaml descriptions).
4. **Что делать** — action cards with ₸ and deadline; buttons state their effect («Сформировать обращение → docx»).
5. **Данные** — the detailed table, collapsed by default.

Global rules (apply to EVERY screen): no naked numbers (label+unit+period+source popover обязательны); breadcrumbs (Договор / КДУ / МРТ); every empty state = 1 предложение что это + 1 кнопка что сделать; no dead ends (every page links its next action); «данные демо» footer badge; role switch visibly changes nav+scope. Passport pattern applies to: contract line, доктор, отделение, дефект, возражение, заявка, клиника (city view).
