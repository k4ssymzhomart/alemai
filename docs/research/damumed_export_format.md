## 1. Что такое Damumed (LIKELY, по сниппетам dmed.kz/агрегаторов)

- **КМИС Damumed** — комплексная медицинская информационная система, вендор **ТОО «Центр Информационных Технологий «ДАМУ»** (Караганда); сайты dmed.kz, damumed.kz; техподдержка support@cit-damu.kz. (LIKELY)
- Развертывание — **региональные облачные инстансы** вида `[регион].dmed.kz`: ast.dmed.kz (Астана), ala.dmed.kz (Алматы), pvd.dmed.kz (Павлодар), zko/akt/atr/vko и т.д. (LIKELY — из агрегаторов lichniy-kabinet.info, spravke.livejournal.com; для Астаны сверить)
- Модули (LIKELY): регистратура (расписания, журналы вызовов на дом, электронные больничные, забор материалов), кабинет врача (ЭМК, списки пациентов, журнал беременных, паспорт здоровья), **медицинская статистика и «ситуационный центр»** (отчеты по отделениям и врачам для администрации), аптека/склад ЛС.
- Интеграция с 1С (CONFIRMED, D4): сервис «1С-Рейтинг: Интеграция с DAMUMED» обменивается данными о количестве и стоимости ЛС/ИМН; интеграция 1С напрямую с ФСМС на 2023 г. была только «в планах» (D4) — то есть финансовые данные ОСМС в бухгалтерию попадали через Damumed/ЕПС, вручную или полуавтоматически.
- Мобильное приложение Damumed (пациентское) существует в Google Play (LIKELY).

## 2. Нормативный мост: что Damumed ОБЯЗАН уметь (CONFIRMED, из D5/D6)

Это ключ к реконструкции выгрузок — требования к любой МИС поставщика ГОБМП/ОСМС:

1. **МИС интегрирована с ИС «ЕПС»** (Единая платежная система ФСМС). Поставщик обеспечивает в МИС «ввод и подтверждение данных согласно первичной медицинской документации (приказ № ҚР ДСМ-175/2020) об оказанных медицинских услугах» — **персонифицированно, в течение 3 рабочих дней с момента оказания** (п.69 Правил оплаты, ред. 2026). Счёт-реестр формируется в ЕПС автоматизированно из этих данных.
2. Реестры-приложения к счёт-реестру (реестр услуг ПМСП, реестры КДУ, реестр НМП-4, реестр детей 6–17) «формируются **на основании данных, внесенных поставщиком в МИС**» — дословные примечания к формам (D6). Значит, в МИС для каждой услуги есть минимум: **полный код услуги, наименование, стоимость (тариф), количество, сумма, ИИН пациента**.
3. Данные прикрепления берутся из портала **«РПН»**, стимкомпонент — из **«ДКПН»**, случаи и услуги — из МИС→ЕПС. Damumed как МИС держит локальные копии/интерфейсы к этим регистрам (INFERRED).
4. Учетная форма **025/у «Талон амбулаторного пациента»** (приказ № ҚР ДСМ-175/2020) — первичный документ амбулаторного случая; заполняется в МИС (LIKELY). Поля талона: ИИН, ФИО, дата рождения, пол, прикрепление/участок, даты обращения, цель/вид посещения, диагноз по МКБ-10, оказанные услуги, специалист (LIKELY — состав по сниппетам, официальный бланк не выкачан).

## 3. ЭРСБ / АПП порталы (LIKELY)

- **ЭРСБ (Электронный регистр стационарных больных)** — централизованная БД **пролеченных случаев** для финансирования стационарной и стационарозамещающей помощи (сниппет D7 — LIKELY). Исторически: ввод случая госпитализации (ИИН, даты поступления/выписки, диагнозы МКБ-10, операции МКБ-9, КЗГ, исход, профиль койки) — поля INFERRED из логики КЗГ-оплаты в Правилах (главы про стационар, линейную шкалу, летальные исходы, повторные госпитализации — CONFIRMED, что такие атрибуты нужны для расчета).
- **Направление АПП** в ЕПС: амбулаторные случаи/услуги (посещения, обращения, КДУ, скрининги). Термин «ПС АПП» встречается в описании ЕПС (сниппет, LIKELY).
- Старые порталы МЗ РК (ЭРСБ, ЭРДБ, РПН на eisz.kz) мигрировали в контур ФСМС/ЕПС и НИЦРЗ; актуальную топологию систем на 2026 г. подтвердить не удалось (UNKNOWN — вопрос менторам).

## 4. Реконструкция выгрузок Damumed — ВЕСЬ РАЗДЕЛ INFERRED

Damumed — веб-система; стандартный паттерн таких МИС: экранные журналы/отчеты с кнопкой **«Экспорт в Excel» (XLSX, реже CSV)**. Что почти наверняка может выгрузить статистик поликлиники:

### 4.1 «Реестр оказанных услуг» за период (для сверки со счёт-реестром / прил. 3–6)
Ожидаемые колонки (ядро = официальные колонки реестров + персонификация, из которой они агрегируются):

| Колонка (рус) | Обоснование |
|---|---|
| № п/п | форма реестра (CONFIRMED в форме, INFERRED в выгрузке) |
| ИИН пациента | обязателен для персонифицированного учета и прил.7–8 |
| ФИО пациента | учетная форма 025/у |
| Дата рождения / возраст | 025/у + возрастные реестры (дети 6–17) |
| Пол | 025/у + половозрастная структура прил.1 |
| Участок / подразделение | привязка к РПН и участкам ПМСП |
| Дата оказания услуги | требование ввода в 3 раб. дня |
| Полный код услуги | колонка формы реестра |
| Наименование услуги | колонка формы реестра |
| Количество | колонка формы реестра |
| Тариф (стоимость услуги, тенге) | колонка формы реестра |
| Сумма, тенге | колонка формы реестра |
| Источник финансирования (ГОБМП/ОСМС) | шапка счёт-реестра; в правилах разведены источники |
| Врач/специалист (код, ФИО) | 025/у, расчет нагрузки |
| Диагноз МКБ-10 | 025/у; нужен для КДУ по показаниям |
| Номер направления | КДУ по направлению, онкоплан (КТ/МРТ/ПЭТ) |
| Статус передачи в ЕПС / ошибки | цикл исправления некорректного счёт-реестра |

### 4.2 Другие типовые отчеты (INFERRED)
- «Отчет по посещениям/обращениям» по врачам и участкам (ситуационный центр — LIKELY, что модуль есть; состав колонок INFERRED).
- «Скрининги» по видам (РМЖ, РШМ, КРР) — под прил.3 формы счёт-реестра.
- «Прикрепленное население» (динамика, половозрастная структура) — под прил.1; первоисточник РПН.
- «Диспансерный учет», «вызовы на дом / неотложная помощь 4 категории» (под прил.8, с ИИН).
- Форматы: XLSX (основной), печатные формы PDF; выгрузка CSV не подтверждена (UNKNOWN).

### 4.3 Чего в Damumed-выгрузке может НЕ быть (важно для IGERIM, INFERRED)
- Итоговых сумм «к оплате» после мониторинга Фонда (снятия/штрафы живут в протоколе исполнения договора в ЕПС, не в МИС).
- Поправочных коэффициентов КПН и расчета подушевой части (считается на стороне ЕПС).
- Статусов оплаты счёт-реестра (акт оказанных услуг, иные выплаты/вычеты — в ЕПС).
- => IGERIM должен принимать НА ВХОД «сырой» реестр услуг из МИС и опционально счёт-реестр/протокол из ЕПС, и находить расхождения между ними.
